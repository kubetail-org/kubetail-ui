# Kubetail-UI

Kubetail-UI is the fun, new, easy way to add Kubetail shared styles and React components to your Kubetail-related apps and projects.

## Quickstart

To use Kubetail-UI, first install the Kubetail-UI JavaScript library and its required dependencies:

```console
pnpm install @kubetail/ui @base-ui/react react react-dom tailwindcss
```

Assuming you've already configured your project to use Tailwind v4, the next step is to import the Kubetail-UI stylesheet into your main CSS file:

```css
@import 'tailwindcss';

@import '@kubetail/ui';
@source '../node_modules/@kubetail/ui/dist/**/*.js';

@custom-variant dark (&:where(.dark, .dark *));
```

Now you can use all of Kubetail-UI's custom styles with Tailwind's utility classes and import components from `@kubetail/ui`:

```tsx
import { Button } from '@kubetail/ui/elements/button';

export default function Page() {
  return (
    <html>
      <body>
        <h1 className="text-heading-4xl">My Cool Project</h1>
        <div className="text-2xl">This is a box with color &quot;bg-error-strong&quot;:</div>
        <div className="h-[100px] w-[100px] bg-destructive" />
        <Button variant="outline" size="lg">Click me</Button>
      </body>
    </html>
  );
}
```

### Portals

Kubetail-UI is built on [Base UI](https://base-ui.com/), which uses portals for components that render popups (e.g. `DropdownMenu`, `Popover`, `Dialog`). For these to position correctly above the rest of the page, wrap your app in a root element that creates its own stacking context:

```tsx
// layout.tsx
<body>
  <div className="root">
    {children}
  </div>
</body>
```

```css
/* styles.css */
.root {
  isolation: isolate;
}
```

This ensures popups always appear above the page contents and any `z-index` in your styles won't interfere with them.

### iOS 26+ Safari

Starting with iOS 26, Safari allows content beneath the UI chrome to be visible. Backdrops such as those used by dialogs must use `position: absolute` instead of `position: fixed` to cover the entire visual viewport. For this to work after the page is scrolled, add the following to your global styles:

```css
/* styles.css */
body {
  position: relative;
}
```

## Fonts

When you import the `@kubetail/ui` css file it automatically applies the following font styles:

```css
@layer base {
  body {
    @apply antialiased;
    font-feature-settings: "rlig" 1, "calt" 1;
  }
}
```

However, this library doesn't load font files for you or set a custom default font family. Picking fonts and ensuring the files are available in the browser is left up to you (though for custom fonts we recommend using variable fonts from FontSource).

## Light/Dark Mode

This library is dark-mode ready but you must choose how you want Tailwind to detect the mode and configure it for your project ([https://tailwindcss.com/docs/dark-mode](https://tailwindcss.com/docs/dark-mode)).
